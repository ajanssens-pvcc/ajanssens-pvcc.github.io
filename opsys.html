Operating Systems Terminology
Operating System The software that controls the overall operation of a computer.
Job Queue In a batch processing system, the job queue is where jobs reside in mass sotrage to wait for execution.
FIFO A system where jobs are removed from the queue in the order which they arrived.
Interactive Processing The ability for a program being executed to carry on a dialog with the user through remote terminals.
Time-Sharing The ability for an operating system to provided service to multiple users at the same time.
Multiprogramming A technique in which time is divided into intervals and the execution of each job is restricted to only one interval at a time.
Application Software The programs for performing tasks particular to the machine's utilization.
System Software The programs for performing tasks that are common to a computer system in general.
Utility Software Software units that extend or customize the capabilities of the operating system.
Graphical User Interface Objects that can be manipulated, such as files and programs, are represented pictorally on the display as icons.
Shell A user interface that communicated with users through textual messages using a keyboard and a monitor screen.
Kernel The software components that perform the very basic functions required by the computer installation.
Device Driver The software units that communicate with the controllers and peripheral devices to carry out operations on the peripheral devices attached to the machine. They translate generic requests into technical steps.
File Manager A software unit that coordinates the use of the machine's mass storage facilities.
Memory Manager The component of an operating system that coordinates the use of the machine's main memory.
Virtual Memory A memory management technique where secondary memory can be used as if it were a part of the main memory.
Boot Strapping The procedure that transfers the operating system from mass storage (where it is permanently stored) into main memory (Which is essentially emoty when the machine is first turned on).
ROM A small portion of a computer's main memory where the CPU expects to find its initial program that is constructed from nonvolatile memory cells. It can be read but not altered i.e. Read-Only Memory.
Deadlock A problem in resource allocation in which two or more processes are blocked from progressing because each is waiting for a resource that is allocated to another.
Spooling The technique of holding data in a mass storage system or a driver for output at a later but more convenient time.
Super User (Administrator) A user profile type that has highly privelaged access to operating systems and can perform tasks such as altering operating system settings, modify software packages, and adjust privelages to other users.
Multi-Factor Authentication A way to secure weak passwords by requiring more than one piece of evidence to gain access.
